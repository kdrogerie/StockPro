🗄️ Supabase
Slaat al je artikelen, GTIN's en mutaties veilig op. Gratis tot 500MB data.
Deel 1 — Supabase (database)
1

Maak gratis Supabase account aan

Ga naar supabase.com → "Start your project" → Gratis account aanmaken via GitHub of e-mail. Maak daarna een nieuw project aan en onthoud het wachtwoord.

2

Maak de database tabellen aan

Ga in Supabase naar SQL Editor → "New query" → plak het onderstaande script en klik op "Run":

-- Artikelen tabel CREATE TABLE artikelen ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, sku TEXT UNIQUE NOT NULL, naam TEXT NOT NULL, beschrijving TEXT, voorraad INTEGER DEFAULT 0, min_voorraad INTEGER DEFAULT 5, created_at TIMESTAMP DEFAULT NOW() ); -- GTIN's tabel (meerdere per artikel) CREATE TABLE gtins ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, artikel_id UUID REFERENCES artikelen(id) ON DELETE CASCADE, gtin TEXT NOT NULL, channels TEXT[] DEFAULT '{}', landen TEXT[] DEFAULT '{}', created_at TIMESTAMP DEFAULT NOW() ); -- Mutaties tabel CREATE TABLE mutaties ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, artikel_id UUID REFERENCES artikelen(id), artikel_naam TEXT, type TEXT NOT NULL, aantal INTEGER NOT NULL, gtin TEXT, kanaal TEXT, notitie TEXT, created_at TIMESTAMP DEFAULT NOW() ); -- Toegang instellen ALTER TABLE artikelen ENABLE ROW LEVEL SECURITY; ALTER TABLE gtins ENABLE ROW LEVEL SECURITY; ALTER TABLE mutaties ENABLE ROW LEVEL SECURITY; CREATE POLICY "Alles toestaan" ON artikelen FOR ALL USING (true) WITH CHECK (true); CREATE POLICY "Alles toestaan" ON gtins FOR ALL USING (true) WITH CHECK (true); CREATE POLICY "Alles toestaan" ON mutaties FOR ALL USING (true) WITH CHECK (true);
3

Kopieer je verbindingsgegevens

Ga in Supabase naar Project Settings → API. Kopieer de Project URL en de anon public key. Vul deze in bij "Instellingen" in deze app.

Deel 2 — GitHub Pages (hosting)
4

Maak gratis GitHub account aan

Ga naar github.com → "Sign up" → Gratis account aanmaken. Bevestig je e-mailadres.

5

Maak een nieuw repository aan

Klik op "New repository" → geef het een naam (bijv. voorraad) → kies Public → klik "Create repository".

6

Upload het HTML-bestand

Klik in je repository op "uploading an existing file" → sleep het bestand voorraad-app.html hierin → hernoem het naar index.html → klik "Commit changes".

7

Zet GitHub Pages aan

Ga naar Settings → Pages in je repository → kies bij "Branch" de optie main → klik "Save". Na 1 minuut is je app live op:

https://JOUWGEBRUIKERSNAAM.github.io/voorraad/

Dit is jouw persoonlijke link, te gebruiken op elke pc, tablet of telefoon.

8

App bijwerken in de toekomst

Wil je een nieuwe versie uploaden? Ga terug naar je GitHub repository → klik op index.html → klik het potlood-icoontje (✏️) → of upload een nieuw bestand. GitHub Pages werkt automatisch bij.
